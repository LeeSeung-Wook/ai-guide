# 회원 인증 시스템 핵심 개념 정리 (BCrypt & Salting)

이 문서는 Spring Boot 환경에서 안전한 회원 가입 및 인증 시스템을 구축하기 위해 사용되는 기술과 원리를 정리한 것입니다.

---

## 1. spring-security-crypto (라이브러리)
- **정의**: 스프링 시큐리티(Spring Security)에서 제공하는 암호화 및 비밀번호 일치 확인을 위한 독립적인 모듈입니다.
- **특징**:
    - 전체 스프링 시큐리티 프레임워크를 도입하지 않고도, 암호화에 필요한 핵심 기능만 선택적으로 사용할 수 있습니다.
    - `build.gradle`에 `implementation 'org.springframework.security:spring-security-crypto'` 의존성을 추가하여 사용합니다.

## 2. BCryptPasswordEncoder (도구)
- **정의**: 비밀번호를 안전하게 암호화하기 위해 가장 널리 사용되는 표준 클래스입니다.
- **핵심 기능**:
    - **encode(rawPassword)**: 원문 비밀번호를 암호화된 해시 문자열로 변환합니다.
    - **matches(rawPassword, encodedPassword)**: 사용자가 입력한 비밀번호와 DB에 저장된 암호화된 비밀번호가 일치하는지 검증합니다. (해시 값은 복호화가 불가능하므로 이 메서드를 통해서만 비교 가능)

## 3. 해시 암호화와 솔팅(Salting) (원리)
### 해시(Hash) 암호화
- **단방향성**: 원문을 암호문으로 바꿀 수는 있지만, 암호문을 다시 원문으로 되돌리는 것(복호화)은 불가능합니다.
- **고정 길이**: 입력 데이터의 길이에 상관없이 항상 일정한 길이의 복잡한 문자열을 생성합니다.

### 솔팅(Salting)
- **개념**: '소금(Salt)을 치다'는 뜻으로, 비밀번호 암호화 시 내부에 랜덤한 데이터를 추가하여 결과값을 매번 다르게 만드는 기술입니다.
- **효과**: 똑같은 "1234" 비밀번호라도 사용자마다 저장되는 암호문이 달라지므로, 해커가 미리 계산해둔 암호표(Rainbow Table)를 이용해 비밀번호를 유추하는 것을 원천 차단합니다.
- **BCrypt의 강점**: BCrypt 알고리즘은 솔팅(Salting) 과정을 자동으로 처리하며, 생성된 해시 값 내부에 소금 정보가 포함되어 있어 별도의 소금 저장 공간이 필요하지 않습니다.

## 4. 회원가입 구현 흐름 (UserService.join)
1. **입력 데이터 수신**: 사용자가 가입 폼에 입력한 ID, 비밀번호, 이메일 등을 `JoinDTO`로 전달받습니다.
2. **비밀번호 암호화**: `BCryptPasswordEncoder.encode()`를 사용하여 비밀번호를 해싱합니다.
3. **엔티티 변환**: 암호화된 비밀번호를 포함하여 `User` 엔티티 객체를 생성합니다.
4. **DB 저장**: `userRepository.save(user)`를 호출하여 영속성 컨텍스트에 저장하고 트랜잭션을 완료합니다.
